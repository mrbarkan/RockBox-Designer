# Font Workspace

Fonts is a dedicated, lazy-loaded Pulp-style workspace over the same canonical package bytes used by import and export. It replaces the old import-only modal; it does not create a second font store.

## Included in the theme

Every `.fnt` under the package's `.rockbox/fonts/` directory is inventoried by exact archive path, owner, SHA-256, byte size, and known CFG or `%Fl` references. Valid RB12 files expose:

- height, ascent, maximum width, depth, declared character range, glyph slots, and file bytes;
- actual 1-bit BDF or 4-bit antialiased glyph pixels;
- Rockbox advance-width measurement and multi-line height for ordinary text;
- code points outside the declared range and probable aliases of the default glyph;
- a 48-cell paged glyph grid, current-project string samples, and same-string comparison across packaged fonts.

Imported bytes remain exact. Replacement validates the complete RB12 bitmap/offset/width layout. Rename updates only references that resolve to the selected exact path. Delete refuses current or otherwise referenced fonts. Selecting the UI font performs a minimum-change `font:` CFG update when an imported CFG exists.

## Rockbox fonts package

Rockbox distributes its font collection separately from the firmware. The workspace includes a metadata-only catalog of the 88 BDF names present at upstream commit `078a506dfd0deb18165a3ed80c7fcbdb3afb0d31`.

No BDF, generated FNT, or third-party font license is bundled. Referencing a catalog entry without importing bytes creates an explicit dependency on the separate Rockbox fonts package installed on the player. Importing the actual FNT makes the theme self-contained and enables glyph preview.

Run `npm run font:catalog:verify` to verify catalog structure. Set `ROCKBOX_SOURCE_DIR` to the pinned checkout to compare all 88 names directly with its `fonts/*.bdf` files.

## Conversion and authority

Existing FNT import works without a helper. TTF/OTF/TTC conversion uses the existing origin-checked loopback helper, the pinned external `convttf`, and an explicit pixel size and glyph range. The browser revalidates the returned RB12 bytes before adding them. Input-font conversion and redistribution remain subject to the source font's license.

RB12 bytes, header metrics, glyph pixels, advance widths, and declared ranges are exact. The browser does not claim full firmware text layout for bidirectional text, combining-mark placement, fallback-font chains, or firmware UI. Those remain external Level C acceptance concerns.

## Browser acceptance

A real `14-Terminus.fnt` generated by the pinned upstream `convbdf` was imported through the visible workflow. The workspace reported 14-pixel height, 12-pixel ascent, 8-pixel maximum width, 65,534 slots, and a 152,800-byte file; rendered recognizable Terminus glyphs; measured `Computer Love 🙂` at 120 pixels; reported U+1F642 outside the declared range with U+FFFD fallback; updated the project UI font; opened Play; and emitted zero application-console errors.
