# Phase 2 WPS Editor

Phase 2 is the first targeted dogfood milestone. The lossless WPS document is still authoritative; semantics, pixels, layer rows, inspector fields, and interaction overlays are projections over that source.

## Supported workflow

1. Import a theme ZIP.
2. Keep `SOURCE_RENDER` enabled on the WPS tab.
3. Use the right panel to inspect global preloads, viewports, elements, conditionals, branches, source-only nodes, and unsupported preserved nodes.
4. Select a supported viewport or element. Drag/resize viewport overlays or edit the safe fields in the source-linked inspector.
5. For a conditional, choose `AUTO` to follow simulation state or select a numbered branch to preview it explicitly.
6. Open `SOURCE_EDITOR`, edit WPS text, and apply it. Valid source updates the preview. Invalid source remains in the document, shows line/column diagnostics, and leaves the last valid preview visible with a stale badge.
7. Export the ZIP. The lossless document and binary package store remain the source of the export.

## Interpreted subset

- Viewports: `%V`, `%Vl`, `%Vi`, `%Vd`
- Foreground/background colors and alignment: `%Vf`, `%Vb`, `%al`, `%ac`, `%ar`
- Font-slot loading and selection approximation: `%Fl`, viewport font slots, `%Fn`
- Metadata and state text: title, artist, album, track/time, codec/bitrate, battery, volume, playback, repeat, shuffle, and clock tags listed in the compatibility matrix
- Images: `%x`, `%xl`, `%xd`, compact frame-count syntax, numeric/state-driven frames, and Rockbox magenta transparency
- Album art: `%Cl`, `%Cd`
- Track and volume bars: `%pb`, `%pv`, including image, slider, and backdrop assets
- Rectangles and touch regions: `%dr`, `%T`
- Scrolling-text approximation: `%s`
- Automatic conditional state: `%mp`, `%mm`, `%bl`, `%pv`, `%ps`, `%mh`, `%bc`, `%bp`, `%bu`, `%lh`, `%C`, `%mv`, metadata presence, and the evidenced `%if`/`%and`/`%or` + `%St` expression subset
- Conditional viewport activation: `%Vl` definitions render only when the active source path enables their label through `%Vd`; false one-branch conditionals render nothing

Every render operation links back to its CST node ID and source span. Constructs outside the subset remain in source and appear as unsupported or source-only layers.

## Rendering contract

- The canvas backing store is exactly the selected device's native resolution.
- Coordinates and sizes are rounded to integer pixels.
- Viewports establish explicit clipping.
- Negative viewport coordinates and dimensions use Rockbox's screen-relative rules.
- CSS scales the finished canvas and requests pixelated image rendering; CSS layout does not define Rockbox pixel positions.
- Bitmap interpolation is disabled.
- `tests/golden/phase2-wps.ppm` is a deterministic 320Ă—240 pixel golden generated by `npm run visual:golden:generate` and verified by `npm run test:visual`.


- The private local AMusicPod and Adwaitapod fixtures each completed a minimum-change viewport edit, semantic re-projection, export, and exact re-import without losing asset hashes.
- A separately supplied wrapped Adwaitapod 3.3 ZIP imported WPS/SBS/FMS with zero package diagnostics and was visually compared beside its 320Ă—240 WPS reference. The comparison verified the active Player/info/playtime viewport set, album-art geometry, text rows, state sprites, image-backed progress bar, and transparent bitmap key. The private ZIP and screenshots are not committed.

- Reports contain hashes and outcomes but no private theme source or files.

## Dogfood boundary

Dogfood the supported WPS subset now, especially viewport placement, basic text/image composition, common playback conditions, bars, and source round trips. Expect unsupported-layer markers on sophisticated themes.

Do not use the Phase 2 preview as a pixel-perfect claim for Rockbox `.fnt` glyphs, condition operands outside the evidenced evaluator, gradients, translated strings, every sprite orientation, sleep-timer state, or SBS/FMS/menu behavior. Those limitations stay preserved and visible rather than being silently flattened.
